# Text Diff Tool 设计文档

**日期**: 2026-04-29
**状态**: 待实现

## 概述

新增一个纯文本对比工具，使用 Google 的 `diff-match-patch` 库计算字符级差异，以并排视图展示文本变化。

## 功能需求

| 功能 | 说明 |
|------|------|
| 并排对比 | 左右两列分别显示原始文本和修改后文本 |
| 字符级高亮 | 在同一行内高亮具体哪些字符发生了变化 |
| 行号显示 | 左右两侧都显示行号 |
| 无语法高亮 | 仅纯文本，不做代码语法着色 |
| 无导出 | 仅页面内对比，不导出差异报告 |

## 核心逻辑

### 差异计算

使用 `diff_match_patch.diff_main(oldText, newText)` 获取差异数组。
每个差异项格式：`[type, text]`
- `type = 0`: 相同内容
- `type = -1`: 删除的内容（仅在原始文本中存在）
- `type = 1`: 新增的内容（仅在修改文本中存在）

### 行对齐算法

1. 将差异数组按换行符分割成行
2. 维护两个行号计数器（originalLine, modifiedLine）
3. 遍历差异项：
   - `type = 0`: 两侧行号都递增，行内容相同
   - `type = -1`: 仅 originalLine 递增，右列对应位置留空
   - `type = 1`: 仅 modifiedLine 递增，左列对应位置留空
4. 输出对齐后的行数组，每行包含：行号、内容、差异类型

### 字符级高亮

对于同一行内既有相同又有差异的情况：
- 删除的字符用 `<span class="diff-deleted">` 包裹
- 新增的字符用 `<span class="diff-added">` 包裹

## 样式规范

遵循项目 Cursor 风格（`DESIGN.md`）：

| 元素 | 样式 |
|------|------|
| 删除行背景 | `rgba(207, 45, 86, 0.08)` |
| 新增行背景 | `rgba(31, 138, 101, 0.08)` |
| 删除字符高亮 | `rgba(207, 45, 86, 0.2)` |
| 新增字符高亮 | `rgba(31, 138, 101, 0.2)` |
| 行号列 | 固定宽度 48px，灰色背景 `var(--color-surface-300)` |
| 字体 | `var(--font-mono)` |
| 边框 | `var(--border-primary)` |

## 交互

| 按钮 | 功能 |
|------|------|
| Compare | 计算并显示差异 |
| Clear | 清空两个输入框和结果区域 |
| Swap | 交换左右输入框内容 |
